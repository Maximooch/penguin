                                                                                                Technical Challenges

   1. Language Mismatch

    • Challenge: OpenCode likely Rust/TypeScript vs Penguin Python
    • Solution: Bridge layer via WebSocket/HTTP
    • Risk: Performance overhead, complexity

   2. State Synchronization

    • Challenge: TUI state vs Penguin state
    • Solution: Event-driven synchronization with EventBus
    • Risk: Race conditions, state divergence

   3. Feature Mismatch

    • Challenge: OpenCode features not in Penguin (and vice versa)
    • Solution: Graceful degradation, feature flags
    • Risk: User confusion, incomplete experience

   4. Maintenance Burden

    • Challenge: Keeping bridge in sync with OpenCode updates
    • Solution: Regular git pulls, automated testing
    • Risk: Breaking changes, drift
