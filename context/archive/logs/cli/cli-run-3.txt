   Based on my analysis of the codebase and refactor plan, here's a comprehensive plan for the file picker feature:



   File Picker Feature Implementation Plan



   Context
   The current CLI uses  click.prompt()  for file selection, which is limiting. The refactor plan calls for a rich, interactive file picker using Textual components.


   Current State (Verified from Code)
     • Basic file prompting in  penguin/cli.py  around lines 50-60
     • No dedicated file picker component exists yet
     • UI components module exists at  penguin/utils/ui_components.py  but doesn't contain file picker


   Implementation Strategy


   1. Component Architecture
     • *Location:**  penguin/utils/ui_components.py  (add to existing file)


     • *Core Features:**
     • Tree-based file browser (using Textual's  DirectoryTree )
     • Multi-file selection support
     • Filter by extension/glob patterns
     • Preview pane for selected files
     • Path navigation breadcrumbs
     • Keyboard shortcuts (arrow keys, space to select, enter to confirm)

   5. Implementation Phases


     • *Phase 1: Basic Picker (MVP)**
     • Single file selection
     • Directory tree navigation
     • Basic keyboard controls
     • Integration with existing CLI commands


     • *Phase 2: Enhanced Features**
     • Multi-select with checkboxes
     • File preview pane
     • Filter by extension/pattern
     • Recent files list
     • Favorites/bookmarks


     • *Phase 3: Advanced UX**
     • Search functionality (fuzzy finding)
     • File type icons
     • Syntax-highlighted preview
     • Git status indicators (modified/staged)
     • Quick actions (open, copy path, reveal)


   6. Technical Considerations


     • *Dependencies:**
     • Textual's  DirectoryTree  widget (already available)
     •  pathlib  for path operations
     • Rich for preview formatting


     • *State Management:**
     • Track selected files in reactive list
     • Persist last used directory
     • Handle large directory trees (lazy loading)


     • *Error Handling:**
     • Permission denied on directories
     • Symlink loops
     • Non-existent paths
     • File system changes during selection


   7. Testing Strategy


     • *Unit Tests:**
     • Path filtering logic
     • Selection state management
     • Keyboard event handling


     • *Integration Tests:**
     • CLI command integration
     • Multi-file workflow
     • Cancel/confirm behavior


     • *Manual Testing:**
     • Large directories (performance)
     • Deep nesting (navigation)
     • Edge cases (empty dirs, special chars)


   8. Acceptance Criteria


   ✅ User can navigate directory tree with arrow keys
   ✅ Space bar toggles file selection
   ✅ Enter confirms selection and returns to CLI
   ✅ ESC cancels without selection
   ✅ Preview pane shows file content for selected item
   ✅ Filter pattern reduces visible files
   ✅ Works with both single and multi-select modes
   ✅ Handles permission errors gracefully
